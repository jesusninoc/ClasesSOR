# Seguridad en sistems en red
- En tu Controlador de Dominio Windows, crea una nueva GPO, edítala para establecer una política de contraseñas más estricta (ej. longitud mínima de 14 caracteres) y aplícala al dominio.
- En tu servidor Linux, usa ufw para permitir el tráfico SSH y denegar todo lo demás.
```Bash
sudo ufw status verbose
sudo ufw allow OpenSSH
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw enable
sudo ufw status verbose
ssh usuario@IP_DEL_SERVIDOR
```
- Comprueba que puedes seguir conectándote por SSH, pero que un ping no recibe respuesta.
Por defecto, ufw suele permitir ciertos tipos de ICMP antes de aplicar las reglas “normales”, por eso aunque tengas deny incoming, el ping puede seguir contestando. Para bloquearlo, se toca la configuración previa de ufw.
```Bash
sudo nano /etc/ufw/before.rules
-A ufw-before-input -p icmp --icmp-type echo-request -j ACCEPT
----> -A ufw-before-input -p icmp --icmp-type echo-request -j DROP
sudo ufw reload
ping IP_DEL_SERVIDOR
```
- En tu Windows Server, usa la herramienta de copias de seguridad para hacer un backup completo del estado del sistema.
- En tu servidor Linux, crea un script simple con tar para comprimir tu carpeta /home y usa cron para que se ejecute cada noche.
- En VirtualBox, toma un snapshot de una VM, haz un cambio (ej. borra un archivo) y luego revierte la máquina al estado del snapshot para recuperarlo.
